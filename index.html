1. Introduction & Overview

What Are Custom Actions?

Custom actions in Power Automate Desktop (PAD) let you write your own reusable actions in C# that appear alongside the built-in actions in the PAD designer. You can drag and drop them into any desktop flow, just like native actions such as "Launch Excel" or "Run PowerShell".

They are useful when:

  • The built-in actions do not cover your use case (e.g., 7-Zip compression, custom database operations)
  • You want to wrap complex logic into a single reusable step
  • You want to share functionality across multiple flows and teams
  • You need to integrate with external systems, libraries, or APIs

How It All Fits Together

C# Code → Build DLL → Sign DLLs → Package into CAB → Sign CAB → Upload to Power Automate → Use in Desktop Flows

Every step is covered in this guide with working examples. By the end, you will have built, signed, packaged, and deployed a working custom action.

3. Creating the Project from Scratch

3.1 Create a Class Library Project

1 Open Visual Studio → Click Create a new project

2 Search for "Class Library (.NET Framework)" — make sure you pick the one that says .NET Framework, NOT .NET or .NET Standard

3 Configure the project:

  • Project name: Modules.MyFirstAction
  • Framework: .NET Framework 4.7.2
  • Location: Any folder (e.g., C:\PADActions)

4 Click Create

5 Delete the auto-generated Class1.cs file

3.2 Naming Conventions

Critical: PAD will only recognize your module if the assembly name follows one of these patterns: 
Modules.*           (e.g., Modules.MyFirstAction)
*.Modules.*         (e.g., Contoso.Modules.FileOps)

If your assembly name does not match, PAD will silently ignore your custom actions.

The AssemblyTitle in project properties acts as the module ID. It can only contain alphanumeric characters and underscores, and must start with a letter.

Warning: Do not use reserved keywords as action names or property names (if, else, loop, end, wait, set, etc.). See the full reserved keywords list.

4. Writing Your First Custom Action

We will build a working "Log Event to File" action step by step, explaining every concept as we go.

4.1 The ActionBase Class & Execute Method

Every custom action is a C# class that: (1) inherits from ActionBase, (2) overrides Execute(ActionContext context), and (3) has the [Action] attribute.

Minimum skeleton:

using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.Attributes;
using System;

namespace Modules.MyFirstAction
{
    [Action(Id = "MyAction")]
    public class MyAction : ActionBase
    {
        public override void Execute(ActionContext context)
        {
            // Your logic goes here
        }
    }
}

4.2 The [Action] Attribute

4.3 Input Arguments

Input arguments are values the user provides when configuring the action. Defined as C# properties with [InputArgument]:

[InputArgument(Required = true,
    Description = "Full path to the log file",
    FriendlyName = "Log File Path")]
public string LogFilePath { get; set; }

Default values use [DefaultValue] from System.ComponentModel:

[InputArgument(Required = false, Description = "Log level")]
[DefaultValue("Info")]
public string LogLevel { get; set; }

4.4 Output Arguments

Output arguments are values your action produces. The user can store them in PAD variables:

[OutputArgument(Description = "True if the log was written successfully")]
public bool Success { get; set; }

// Set inside Execute:
Success = true;  // This value becomes available in PAD

4.6 Complete Working Example: File Logger Action

Create LogEventAction.cs in your project:

using System;
using System.IO;
using System.ComponentModel;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.Attributes;

namespace Modules.MyFirstAction
{
    [Action(Id = "LogEventToFile", Order = 1, Category = "Logging",
        FriendlyName = "Log Event To File",
        Description = "Writes a timestamped message to a log file")]
    [Throws("LogWriteError",
        FriendlyName = "Log Write Error",
        Description = "Failed to write the log message to the file")]
    public class LogEventAction : ActionBase
    {
        [InputArgument(Required = true,
            FriendlyName = "Log File Path",
            Description = "Full path to the log file (e.g., C:\\Logs\\app.log)")]
        public string LogFilePath { get; set; }

        [InputArgument(Required = true,
            FriendlyName = "Message",
            Description = "The message to write to the log file")]
        public string Message { get; set; }

        [InputArgument(Required = false,
            FriendlyName = "Log Level",
            Description = "The severity level (Info, Warning, Error)")]
        [DefaultValue("Info")]
        public string LogLevel { get; set; }

        [OutputArgument(FriendlyName = "Success",
            Description = "True if the message was written successfully")]
        public bool Success { get; set; }

        public LogEventAction()
        {
            LogFilePath = string.Empty;
            Message = string.Empty;
            LogLevel = "Info";
            Success = false;
        }

        public override void Execute(ActionContext context)
        {
            try
            {
                string timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
                string logLine = $"[{timestamp}] [{LogLevel}] {Message}" + Environment.NewLine;

                string directory = Path.GetDirectoryName(LogFilePath);
                if (!string.IsNullOrEmpty(directory) && !Directory.Exists(directory))
                    Directory.CreateDirectory(directory);

                File.AppendAllText(LogFilePath, logLine);
                Success = true;
            }
            catch (Exception e)
            {
                if (e is ActionException) throw;
                throw new ActionException("LogWriteError", e.Message, e.InnerException);
            }
        }
    }
}
Try it: Build the project (Ctrl+Shift+B). If it compiles without errors, you have a working custom action. The DLL will be in bin\Release\net472\.

5. Error Handling

5.1 The [Throws] Attribute

The [Throws] attribute declares what errors your action can produce. These appear in the PAD designer's error handling tab:

[Throws("FILE_NOT_FOUND",
    FriendlyName = "File Not Found",
    Description = "The specified file does not exist")]
[Throws("PERMISSION_DENIED",
    FriendlyName = "Permission Denied",
    Description = "Insufficient permissions to access the file")]
public class MyAction : ActionBase { ... }

5.2 Throwing ActionExceptions

The first parameter of ActionException must match a [Throws] ID exactly:

throw new ActionException("FILE_NOT_FOUND", "The file was not found: " + filePath);

Standard pattern:

public override void Execute(ActionContext context)
{
    try
    {
        // Your logic
    }
    catch (Exception e)
    {
        if (e is ActionException) throw;  // Don't double-wrap
        throw new ActionException("GENERAL_ERROR", e.Message, e.InnerException);
    }
}
Note: Always check if (e is ActionException) throw; first. This prevents double-wrapping when you throw an ActionException inside a nested method.

5.3 Working Example: Adding Error Handling

Enhanced File Logger with two specific error types:

[Action(Id = "LogEventToFile", Order = 1, Category = "Logging")]
[Throws("INVALID_PATH", FriendlyName = "Invalid File Path",
    Description = "The log file path is empty or invalid")]
[Throws("WRITE_FAILED", FriendlyName = "Write Failed",
    Description = "Could not write to the log file")]
public class LogEventAction : ActionBase
{
    [InputArgument(Required = true)]
    public string LogFilePath { get; set; }

    [InputArgument(Required = true)]
    public string Message { get; set; }

    [OutputArgument]
    public bool Success { get; set; }

    public override void Execute(ActionContext context)
    {
        try
        {
            if (string.IsNullOrWhiteSpace(LogFilePath))
                throw new ActionException("INVALID_PATH", "Log file path cannot be empty.");

            File.AppendAllText(LogFilePath,
                $"[{DateTime.Now:yyyy-MM-dd HH:mm:ss}] {Message}\r\n");
            Success = true;
        }
        catch (Exception e)
        {
            if (e is ActionException) throw;
            throw new ActionException("WRITE_FAILED",
                $"Failed to write log: {e.Message}", e.InnerException);
        }
    }
}

7. Advanced Features

7.1 Conditional Actions

Conditional actions return true/false for use in "If" blocks (like "If file exists"). Use [ConditionAction]:

[ConditionAction(Id = "CheckFileExists", ResultPropertyName = nameof(Result))]
[Throws("CheckError")]
public class CheckFileExistsAction : ActionBase
{
    public bool Result { get; private set; }

    [InputArgument(Required = true, Description = "Path to check")]
    public string FilePath { get; set; }

    public override void Execute(ActionContext context)
    {
        try { Result = File.Exists(FilePath); }
        catch (Exception e)
        {
            if (e is ActionException) throw;
            throw new ActionException("CheckError", e.Message, e.InnerException);
        }
    }
}

Key differences: use [ConditionAction] not [Action]; Result has private set; do NOT add [OutputArgument] to Result.

7.2 Action Selectors (Enum-based)

Selectors create multiple "flavors" of one action (like "Launch Excel" having "blank document" vs "open file" variants).

Step 1: Define an enum:

public enum LogDestination { File, Console, Both }

Step 2: Base action with enum input:

[Action(Id = "LogEvent")]
public class LogEventAction : ActionBase
{
    [InputArgument, DefaultValue(LogDestination.File)]
    public LogDestination Destination { get; set; }

    [InputArgument(Order = 1)]
    public string FilePath { get; set; }

    [InputArgument(Order = 2)]
    public string Message { get; set; }

    [OutputArgument]
    public bool Success { get; set; }

    public override void Execute(ActionContext context)
    {
        if (Destination == LogDestination.File || Destination == LogDestination.Both)
            File.AppendAllText(FilePath, Message + "\r\n");
        Success = true;
    }
}

Step 3: Create selector classes (NOT marked with [Action]):

public class LogToFile : ActionSelector<LogEventAction>
{
    public LogToFile()
    {
        UseName("LogToFile");
        Prop(p => p.Destination).ShouldBe(LogDestination.File);
        ShowAll();
    }
}

public class LogToConsole : ActionSelector<LogEventAction>
{
    public LogToConsole()
    {
        UseName("LogToConsole");
        Prop(p => p.Destination).ShouldBe(LogDestination.Console);
        ShowAll();
        Hide(p => p.FilePath);  // Not needed for console
    }
}

7.3 Action Selectors (Boolean-based)

For two-option selectors, use a boolean instead of an enum:

[Action(Id = "WatchFolder")]
public class WatchFolderAction : ActionBase
{
    [InputArgument, DefaultValue(true)]
    public bool MoveFile { get; set; }

    [InputArgument(Order = 1)]  public string SourceFolder { get; set; }
    [InputArgument(Order = 2)]  public string DestFolder { get; set; }
    [OutputArgument]            public string DetectedFile { get; set; }

    public override void Execute(ActionContext context) { /* ... */ }
}

public class WatchAndMove : ActionSelector<WatchFolderAction>
{
    public WatchAndMove()
    {
        UseName("WatchAndMoveFile");
        Prop(p => p.MoveFile).ShouldBe(true);
        ShowAll();
    }
}

public class WatchOnly : ActionSelector<WatchFolderAction>
{
    public WatchOnly()
    {
        UseName("WatchFileOnly");
        Prop(p => p.MoveFile).ShouldBe(false);
        ShowAll();
        Hide(p => p.DestFolder);
    }
}

Use dot notation: [Action(Category = "Logging.Write")] creates a tree structure in the sidebar.

8. Certificates: Creating & Managing

Mandatory: ALL DLL files and the CAB file must be signed with a digital certificate trusted by the machine. PAD will refuse to load unsigned custom actions.

8.1 Self-Signed Certificates (Development Only)

Create using PowerShell (run as Administrator):

$certName    = "PADCustomActionDevCert"
$certFolder  = "C:\PADActions\Certs"
$pfxPassword = "YourSecurePassword123!"

if (!(Test-Path $certFolder)) { New-Item -ItemType Directory -Path $certFolder -Force }

$cert = New-SelfSignedCertificate `
    -CertStoreLocation Cert:\CurrentUser\My `
    -Type CodeSigningCert `
    -Subject "CN=$certName" `
    -KeyExportPolicy Exportable `
    -KeySpec Signature `
    -KeyLength 2048 `
    -KeyAlgorithm RSA `
    -HashAlgorithm SHA256

$secPwd = ConvertTo-SecureString -String $pfxPassword -Force -AsPlainText
Export-PfxCertificate -Cert $cert -FilePath "$certFolder\$certName.pfx" -Password $secPwd
Export-Certificate -Cert $cert -FilePath "$certFolder\$certName.cer" -Type CERT -Force

Write-Host "Created: $certFolder\$certName.pfx and .cer" -ForegroundColor Green
FileContainsUsed For
.pfxCertificate + private keySigning DLLs and CAB files
.cerCertificate only (public)Installing on machines that need to trust the cert
Warning: Self-signed certificates are for development only. For production, use a certificate from your organization's Certificate Authority.

8.2 Enterprise Certificates (Production)

Request a Code Signing Certificate from your IT/PKI team. It must have an exportable private key and code signing capabilities.

8.3 Installing to Trusted Root Store

Required on every machine that will author or run flows with your custom action. Run as Administrator:

Import-Certificate `
    -FilePath "C:\PADActions\Certs\PADCustomActionDevCert.cer" `
    -CertStoreLocation "Cert:\LocalMachine\AuthRoot"

Verify in certlm.msc under Trusted Root Certification Authorities → Certificates.

8.4 Working Example: Certificate Installation Script

Save as install-cert.ps1 and give to end users:

if ([Environment]::UserInteractive) {
    Clear-Host
    Write-Host "=== Certificate Installation for PAD Custom Actions ===" -ForegroundColor Cyan
    $CertPath = Read-Host "Enter the full path to the .cer file"
}

try {
    $identity  = [Security.Principal.WindowsIdentity]::GetCurrent()
    $principal = New-Object Security.Principal.WindowsPrincipal($identity)
    if (!$principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)) {
        throw "This script requires administrator privileges."
    }
    if (!(Test-Path $CertPath)) { throw "Certificate file not found: $CertPath" }

    Import-Certificate -FilePath $CertPath -CertStoreLocation "Cert:\LocalMachine\AuthRoot" | Out-Null
    Write-Host "Certificate installed successfully!" -ForegroundColor Green
}
catch { Write-Host "Error: $_" -ForegroundColor Red; exit 1 }

if ([Environment]::UserInteractive) { Read-Host "Press Enter to exit" }

9. Signing DLLs

9.1 Why Signing is Required

Every DLL in your package must be signed: your action DLL plus all dependency DLLs. Exclude Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.dll (already signed by Microsoft).

9.2 Using SignTool

signtool sign /f "MyCert.pfx" /p YourPassword /fd SHA256 "Modules.MyFirstAction.dll"
FlagMeaning
/fPath to .pfx certificate
/pPFX password
/fd SHA256SHA-256 digest algorithm

9.3 Working Example: Batch Signing Script

# sign-dlls.ps1
param(
    [Parameter(Mandatory)] [string]$ReleaseFolder,
    [Parameter(Mandatory)] [string]$PfxPath,
    [Parameter(Mandatory)] [string]$PfxPassword
)

$dlls = Get-ChildItem $ReleaseFolder -Filter *.dll
Write-Host "Found $($dlls.Count) DLL(s) to sign" -ForegroundColor Cyan

foreach ($dll in $dlls) {
    Write-Host "Signing $($dll.Name)..." -ForegroundColor Gray
    & signtool sign /f $PfxPath /p $PfxPassword /fd SHA256 $dll.FullName
    if ($LASTEXITCODE -ne 0) { Write-Host "FAILED" -ForegroundColor Red; exit 1 }
}
Write-Host "All DLLs signed." -ForegroundColor Green

10. Packaging into a CAB File

10.1 What is a CAB File?

A Cabinet (.cab) file is a Windows archive format. PAD requires all custom action DLLs packaged into a single signed .cab for upload.

10.2 The makeCabFile.ps1 Script Explained

# makeCabFile.ps1
param(
    [ValidateScript({Test-Path $_ -PathType Container})]
    [string]$sourceDir,     # Folder with compiled DLLs

    [ValidateScript({Test-Path $_ -PathType Container})]
    [string]$cabOutputDir,  # Where to save the .cab

    [string]$cabFilename    # Name of the .cab file
)

# Build a DDF (Diamond Directive File) telling makecab what to include
$ddf = ".OPTION EXPLICIT
.Set CabinetName1=$cabFilename
.Set DiskDirectory1=$cabOutputDir
.Set CompressionType=LZX
.Set Cabinet=on
.Set Compress=on
.Set CabinetFileCountThreshold=0
.Set FolderFileCountThreshold=0
.Set FolderSizeThreshold=0
.Set MaxCabinetSize=0
.Set MaxDiskFileCount=0
.Set MaxDiskSize=0
"

$ddfpath = ($env:TEMP + "\customModule.ddf")
$sourceDirLength = $sourceDir.Length

# Add DLLs (except the SDK DLL)
$ddf += (Get-ChildItem $sourceDir -Filter "*.dll" |
    Where-Object { (!$_.PSIsContainer) -and
        ($_.Name -ne "Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.dll") } |
    Select-Object -ExpandProperty FullName |
    ForEach-Object { '"' + $_ + '" "' + ($_.Substring($sourceDirLength)) + '"' }
) -join "`r`n"

$ddf | Out-File -Encoding UTF8 $ddfpath
makecab.exe /F $ddfpath
Remove-Item $ddfpath

10.3 Working Example: Building & Signing the CAB

# Build → Sign DLLs → Create CAB → Sign CAB
dotnet build Modules.MyFirstAction.csproj -c Release

Get-ChildItem ".\bin\Release\net472" -Filter *.dll | ForEach-Object {
    signtool sign /f "MyCert.pfx" /p "Password" /fd SHA256 $_.FullName
}

.\makeCabFile.ps1 ".\bin\Release\net472" ".\Package" "MyFirstAction.cab"

signtool sign /f "MyCert.pfx" /p "Password" /fd SHA256 ".\Package\MyFirstAction.cab"
Do not forget to sign the CAB! Unsigned CAB files will be rejected by PAD.

11. Deployment & Usage

11.1 Uploading to Power Automate Portal

  1. Go to make.powerautomate.com
  2. Navigate to Desktop flows → Custom actions
  3. Click Upload, provide name/description, select the signed .cab file

11.2 Adding to Desktop Flows

  1. Open PAD, create/edit a desktop flow
  2. Click Assets Library in the toolbar
  3. Find your module, click Add
  4. Search for your action in the Actions panel, drag into your flow
  5. Configure inputs and use output variables in later steps

11.3 Runtime Machine Setup

Every machine running flows with your custom action needs: (1) the signing certificate in Trusted Root, (2) Power Automate Desktop installed.

Enterprise Tip: Bundle the .cer with your PAD installer via SCCM/Intune for automatic certificate deployment.

11.4 Updating & Redeploying

Modify code → Rebuild → Re-sign DLLs → Re-create & re-sign CAB → Upload (same name updates the existing action).

Breaking changes: Adding new required inputs or removing existing ones will break existing flows using the action.

12. Full End-to-End Working Example

Complete walkthrough: building a "Text File Counter" action that counts lines, words, and characters.

Step 1: Create Project

Visual Studio → Class Library (.NET Framework) → Name: Modules.TextTools → Framework: 4.7.2 → Delete Class1.cs → Install SDK NuGet package.

Step 2: Write the Action

Create CountFileStatsAction.cs:

using System;
using System.IO;
using System.Linq;
using System.ComponentModel;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.Attributes;

namespace Modules.TextTools
{
    [Action(Id = "CountFileStats", Order = 1, Category = "TextAnalysis",
        FriendlyName = "Count File Statistics",
        Description = "Counts lines, words, and characters in a text file")]
    [Throws("FILE_NOT_FOUND", FriendlyName = "File Not Found",
        Description = "The specified file does not exist")]
    [Throws("READ_ERROR", FriendlyName = "Read Error",
        Description = "Failed to read the file")]
    public class CountFileStatsAction : ActionBase
    {
        [InputArgument(Required = true, FriendlyName = "File Path",
            Description = "Full path to the text file to analyze")]
        public string FilePath { get; set; }

        [InputArgument(Required = false, FriendlyName = "Count Empty Lines",
            Description = "Include empty lines in the count")]
        [DefaultValue(true)]
        public bool CountEmptyLines { get; set; }

        [OutputArgument(FriendlyName = "Line Count")]
        public int LineCount { get; set; }

        [OutputArgument(FriendlyName = "Word Count")]
        public int WordCount { get; set; }

        [OutputArgument(FriendlyName = "Character Count",
            Description = "Characters excluding newlines")]
        public int CharCount { get; set; }

        public CountFileStatsAction()
        {
            FilePath = string.Empty;
            CountEmptyLines = true;
        }

        public override void Execute(ActionContext context)
        {
            try
            {
                if (!File.Exists(FilePath))
                    throw new ActionException("FILE_NOT_FOUND", $"File not found: {FilePath}");

                string content = File.ReadAllText(FilePath);
                string[] lines = File.ReadAllLines(FilePath);

                LineCount = CountEmptyLines
                    ? lines.Length
                    : lines.Count(l => !string.IsNullOrWhiteSpace(l));

                WordCount = content
                    .Split(new[] { ' ', '\t', '\r', '\n' }, StringSplitOptions.RemoveEmptyEntries)
                    .Length;

                CharCount = content.Replace("\r", "").Replace("\n", "").Length;
            }
            catch (Exception e)
            {
                if (e is ActionException) throw;
                throw new ActionException("READ_ERROR", e.Message, e.InnerException);
            }
        }
    }
}

13. Real-World Example: Download Monitor Action

An advanced production example that monitors the Downloads folder for new files and moves them to a destination. Demonstrates timeouts, polling, file operations, and multiple error types.

using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK;
using Microsoft.PowerPlatform.PowerAutomate.Desktop.Actions.SDK.Attributes;
using System;
using System.Collections.Generic;
using System.ComponentModel;
using System.IO;
using System.Linq;
using System.Threading;

namespace Modules.FileTools
{
    [Action(Id = "WaitForDownload", Category = "FileSystem", Order = 1,
        FriendlyName = "Wait For Download And Move File",
        Description = "Monitors downloads folder for new files and moves them")]
    [Throws("INVALID_DESTINATION", FriendlyName = "Invalid Destination",
        Description = "The destination folder is invalid")]
    [Throws("DOWNLOAD_TIMEOUT", FriendlyName = "Download Timeout",
        Description = "No new file appeared within the timeout")]
    [Throws("MOVE_FAILED", FriendlyName = "File Move Failed",
        Description = "Could not move the downloaded file")]
    public class WaitForDownloadAction : ActionBase
    {
        [InputArgument(Required = true, Description = "Destination folder for the file")]
        public string DestinationFolder { get; set; }

        [InputArgument(Required = false, Description = "Extensions to ignore")]
        [DefaultValue(".crdownload,.download,.part,.tmp")]
        public string ExcludedExtensions { get; set; }

        [InputArgument(Required = false, Description = "Max seconds to wait")]
        [DefaultValue(300)]
        public int? TimeoutSeconds { get; set; }

        [OutputArgument(Description = "Full path of the moved file")]
        public string MovedFilePath { get; set; }

        [OutputArgument(Description = "Name of the downloaded file")]
        public string FileName { get; set; }

        [OutputArgument(Description = "True if successful")]
        public bool Success { get; set; }

        private string DownloadsFolder => Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.UserProfile), "Downloads");

        public WaitForDownloadAction()
        {
            DestinationFolder = string.Empty;
            ExcludedExtensions = ".crdownload,.download,.part,.tmp";
            TimeoutSeconds = 300;
            MovedFilePath = string.Empty;
            FileName = string.Empty;
            Success = false;
        }

        public override void Execute(ActionContext context)
        {
            try
            {
                if (string.IsNullOrEmpty(DestinationFolder))
                    throw new ActionException("INVALID_DESTINATION", "Destination cannot be empty.");

                if (!Directory.Exists(DestinationFolder))
                    Directory.CreateDirectory(DestinationFolder);

                // Snapshot current files
                var existingFiles = GetNonTempFiles();

                // Poll for new file
                int timeout = TimeoutSeconds ?? 300;
                var deadline = DateTime.Now.AddSeconds(timeout);
                string newFile = null;

                while (DateTime.Now < deadline)
                {
                    var currentFiles = GetNonTempFiles();
                    var newFiles = currentFiles.Except(existingFiles).ToList();

                    if (newFiles.Count == 1) { newFile = newFiles[0]; break; }
                    else if (newFiles.Count > 1) {
                        newFile = newFiles.OrderByDescending(f => File.GetLastWriteTime(f)).First();
                        break;
                    }
                    Thread.Sleep(1000);
                }

                if (newFile == null)
                    throw new ActionException("DOWNLOAD_TIMEOUT",
                        $"No new file detected within {timeout} seconds.");

                // Move with collision handling
                string destPath = Path.Combine(DestinationFolder, Path.GetFileName(newFile));
                if (File.Exists(destPath))
                {
                    string noExt = Path.GetFileNameWithoutExtension(newFile);
                    string ext = Path.GetExtension(newFile);
                    int i = 1;
                    do { destPath = Path.Combine(DestinationFolder, $"{noExt}({i}){ext}"); i++; }
                    while (File.Exists(destPath));
                }

                File.Move(newFile, destPath);
                MovedFilePath = destPath;
                FileName = Path.GetFileName(destPath);
                Success = true;
            }
            catch (Exception e)
            {
                if (e is ActionException) throw;
                throw new ActionException("MOVE_FAILED", e.Message, e.InnerException);
            }
        }

        private HashSet GetNonTempFiles()
        {
            var excluded = ExcludedExtensions
                .Split(new[] { ',' }, StringSplitOptions.RemoveEmptyEntries)
                .Select(e => e.Trim().ToLowerInvariant()).ToList();

            return new HashSet(
                Directory.GetFiles(DownloadsFolder)
                    .Where(f => !excluded.Contains(Path.GetExtension(f).ToLowerInvariant()))
                    .Select(f => f.ToLowerInvariant()));
        }
    }
}

Patterns demonstrated: polling with deadline timeout, before/after file snapshots, filename collision handling with (1)/(2) suffixes, nullable inputs with defaults, private helper methods.

14. Troubleshooting & Common Pitfalls

ProblemCauseSolution
Action doesn't appear in PADAssembly name doesn't match Modules.*Rename project so DLL is Modules.Something.dll
"Action is not trusted" errorUnsigned DLLs/CAB or cert not in Trusted RootSign everything, install .cer in Cert:\LocalMachine\AuthRoot
CAB upload failsCAB file not signedRun signtool on the .cab after creating it
Reserved keyword errorsAction/property uses reserved nameRename. Check reserved keywords list
Inputs/outputs missing in designerMissing [InputArgument] or [OutputArgument]Add the attributes to each property
Default values not appliedMissing [DefaultValue] attributePAD reads from the attribute, not the constructor
Generic "ActionError" instead of specificCatch block wraps everything genericallyAdd specific [Throws] + matching ActionException
signtool not foundWindows SDK not installed/in PATHInstall via VS Installer, or use Developer Command Prompt
Wrong framework errorUsing .NET 6/7/8 instead of Framework 4.7.2Set TargetFramework to net472
Module conflictsTwo modules have same AssemblyTitleUse unique AssemblyTitle per module
